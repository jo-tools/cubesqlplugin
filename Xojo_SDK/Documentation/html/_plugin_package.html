Xojo Plugins SDK
Plugin Package

Plugins support two package formats: Zip and Folder

Zip Plugins can now be packaged using Zip-based xojo_plugin packages. These packages are simply Zip files of the plugin package file structure with the extension changed from zip to xojo_plugin.

Folder You can also simply use a folder instead of packaging your plugins as a Zip file. The folder must have a xojo_plugin extension, but is otherwise just a directory using the plugin package file structure.

Plugin Package File Structure (as of 2022r4) Both formats share the same package file structure.

NOTE: The folder names are CASE SENSITIVE! 

+-Plugin Name               // name of your plugin
    +- Build Resources      // includes shared library parts
        +-Linux X86
        +-Linux x86-64
        +-Linux ARM
        +-Linux arm64       // New for 2022r4
        +-iOS Device        // New for 2020r2 (arm64)
        +-iOS Simulator     // New for 2020r2 (x86_64)
        +-Mac arm64         // New for 2020r2
        +-Mac x86-64
        +-Mac Universal     // include x86_64 and arm64 targets as a single binary
        +-Windows
        +-Windows arm64     // New for 2022r2
        +-Windows x86-64
        +-WinUI x86         // New for 2023r2 (Note: DLL name must match the same under Windows)
        +-WinUI x86-64      // New for 2023r2 (Note: DLL name must match the same under Windows x86-64)
        +-WinUI arm64       // New for 2023r2 (Note: DLL name must match the same under Windows arm64)
    +- IDE Resources        // includes pictures/docs for your plugin
        +-Controls Palette  // includes icon you'd see in the toolbar
        +-Help              // plugin docs, must include index.html

The names of the directories inside of the 'Build Resources' directory are fairly self-explanatory. The "Mac x86" and "Mac Universal" folders are new to 2018r3, so will not be recognized in older versions of Xojo.

Platform Categories To provide a way to customize your plugin to fit the OS, you can add your resources inside platform sub folders, for example: 

+IDE Resources
    +-Controls Palette
          128-library-small.tiff
          128-library-large.tiff
        +-Mac Cocoa
        +-Windows
              128-library-small.tiff
              128-library-large.tiff

For example, if you want a different looking picture to appear on the Windows IDE, you could include a picture inside the sub folder "Windows". Xojo will first look inside the platform-specific folder and, if an image is not found, the top level "Controls Palette" folder. Note that the picture filename must match the id number (i.e. the toolbarPICT field of the REALcontrol struct) that you defined for your control.

As of 2016r1, support has been added for higher resolution icons to be used in the Xojo library pane. Plugins should provide a small version which is 16x16 and larger version that is 48x48. In order to support HiDPI, the TIFF files should each have a 72 DPI image and a 144 DPI image inside.

Prior versions of the IDE only looked for a file ending in ".bmp" and a mask file ending in "mask.bmp". For example, "128.bmp" and "128mask.bmp". 2016r1 will fall back to this if the library images are absent in order to maintain some degree of compatibility.

HTML Help You can include HTML documentation that can be referenced from within the IDE. If a plugin includes documentation, a new menu item will appear under the Help menu in the Xojo IDE. Documentation is valid ONLY IF you include a file named "index.html" inside the Help folder. 

+-IDE Resources
    +-Help
          index.html
          pic.jpg
          another.html
          another2.html

If Xojo does not find this one critical file, the menu item will not appear, and thusly no documentation can be referenced. You may include multiple files, but no sub folders inside the Help folder will be recognized.
