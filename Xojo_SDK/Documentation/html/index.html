About the Plugins SDK

The Plugins SDK (software development kit) is for using C/C++ to make your own plugin functions and controls for use with Xojo.

Plugins can be created for use by desktop, iOS, web and console projects.

This folder contains code and documentation to get you started making plug-in functions and controls for Xojo. You'll find the following folders:

  1. Documentation: this folder contains the SDK documentation in HTML format.
  2. Examples: you'll find several subfolders here illustrating how to make Xojo plug-ins.
  3. Glue Code: this folder contains C++ code you must add to your plug-in projects.
  4. Includes: these files are included by the glue code (and most likely by your own plug-in code as well).

Note: The REALxxx prefix for a lot of the functions and types are carried over from Xojo's legacy naming convention.

Prerequisites

Here are some recommended tools utilized by our examples:

  • Xcode 9+
  • Visual Studio 2015 or later
  • Makefile/CMake and gcc/clang (or g++/clang++)

Anatomy of a Xojo plugin

A plugin is built as a shared library on the following platforms:

  • .dylib on macOS and iOS
  • .DLL on Windows
  • .so on Linux/Pi

It exposes a single entry point to both communicate with the Xojo IDE and for accessibility during Runtime.

  • void PluginEntry()

From within the entry point method you can register methods, modules, etc. which will be your building blocks to let the IDE know what your plugin supports. Within the entry point you can also describe what other classes that your plugin depends on which is useful to let our compiler know so certain classes do not get dead stripped.

Where to put your plugins

For testing purposes, you can simply put your plugin (i.e. .dylib, .DLL or .so file) into the Xojo IDE's Plugins folder.

Note: Since the Xojo IDE is built as a 64-bit app you must provide at least a 64-bit target. For example if you built a 32-bit DLL and put it in the IDE's Plugins folder it will not be loaded.

You can find the IDE's Plugins folder wherever you installed Xojo. To support multiple platforms/architectures, and/or to add additional resources like HTML help or Control Icons along with your plugin, we recommend our custom Plugin Package structure. This can either be a flat folder hierarchy or a zipped up folder hierarchy (both with the .xojo_plugin extension). See Plugin Package Format.
